Markdown Rules

Phrase Emphasis

*italic*   **bold**
_italic_   __bold__

Links

Inline:

[desc](http://a.com/ "title")

Reference-style label (titles are optional):

use [desc][id] anywhere else
in doc, define the link:

[id]: http://a.com/ "title"

Images

Inline (titles are optional):

![alt](/path/a.jpg "title")

Reference-style:

![alt][id]

[id]: /path/a.jpg "title"

Headers

Setext-style:

Header 1
========

Header 2
--------

atx-style (closing # are optional):

# Header 1 #

### Header 3 ###

###### Header 6

Lists

Ordered, without paragraphs:

1. Foo
2. Bar

Unordered, with paragraphs:

*   A list item.

    With multiple paragraphs.

*   Bar

You can nest them:

*   Abacus
    * answer
*   Bubbles
    1.  bunk
    2.  bupkis
        * BELITTLER
    3. burper
*   Cunning

Blockquotes

> Email-style angle brackets
> are used for blockquotes.

&gt; And, this work well.

:: This also work well.

> > And, they can be nested.

> ### Header in blockquotes
>
> * You can quote a list.
> * Etc.

Code Spans

`<code>` spans are delimited
by backticks.

You can include literal
backticks like `` `this` ``.

Preformatted Code Blocks

Indent every line of a code block by at least 4 spaces or 1 tab.

This is a normal paragraph.

    This is a preformatted
    code block.

Horizontal Rules

Three or more dashes or asterisks:

---

* * *

- - - -

Manual Line Breaks

End a line with two or more spaces:

Roses are red,   
Violets are blue.

RefDiv reference (extended)

<pre class="rewgt-mdref" 
path="/path/to/ref"></pre>

Anchor for link (extended)

use [desc](#id) anywhere else
in doc, define the anchor:

<a name="id"></a>
